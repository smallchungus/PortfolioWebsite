# Documentation Standards

## ADRs (Architecture Decision Records)
- Max 50 lines
- Format: Context → Decision → Consequences
- No code blocks
- Brief, actionable

## Technical Docs (.md files)
- Max 100 lines (150 for complex topics)
- High-level only
- No implementation code
- Use bullet points

## README Exception
- Can exceed limits
- Include setup instructions
- Badges and project info

## Writing Style
- Concise sentences
- Active voice
- Value-focused
- No fluff
